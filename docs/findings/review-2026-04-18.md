## 🔴 Must fix (resolved)

### commands/build.md:21, commands/remediate.md:27 — `exit 1` would kill persistent Bash shell
**Category:** functionality
**Confidence:** HIGH
**Impact:** Pipeline skills are executed by copying Bash snippets into the
Claude Code Bash tool, which holds a single persistent shell. `exit 1` in a
non-subshell terminates the current shell — every subsequent `git`, `node
scripts/*`, and Postgres query in the session would fail until the shell
restarts.
**Remediation:** Dropped `&& exit 1` from both base-branch guard snippets;
echo of STOP message is the signal, agent halts on seeing it.
**Fix:** applied in commands/build.md and commands/remediate.md.

## 🟡 Should fix (resolved)

### skills/orientation/SKILL.md:59, commands/build.md, commands/remediate.md — `\s` in regex is GNU-only
**Category:** platform-portability
**Confidence:** MEDIUM
**Impact:** `grep -E '^\s*branch:'` and `sed 's/...\s*...'` use GNU-specific
`\s` whitespace class. BSD sed on macOS (default system sed) does not handle
`\s` reliably in all modes. Pipeline must run on all three Claude Code target
platforms (Windows Git Bash, macOS, Linux).
**Remediation:** Replaced `\s*` with `[[:space:]]*` (POSIX character class)
across all three locations. Verified behavioral parity via test.
**Fix:** applied.

### commands/*.md — preflight references `skills/orientation/SKILL.md` as bare relative path
**Category:** cross-file-contract
**Confidence:** HIGH
**Impact:** Every other skill reference in the command files uses the standard
resolution pattern ("`$PIPELINE_DIR/skills/X/SKILL.md` if set, otherwise Glob
`**/pipeline/skills/X/SKILL.md`"). The new orientation references used a bare
relative path, which would resolve against the user's project root when Pipeline
is installed at user scope — the file wouldn't be found.
**Remediation:** Added the standard resolution preamble to all 8 command
preflight references.
**Fix:** applied to build.md, review.md, qa.md, redteam.md, audit.md,
remediate.md, commit.md, finish.md.
